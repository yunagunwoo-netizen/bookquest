## 3. 사고 로그 — Edit 도구 truncation 재발 (중요)

MD에서 경고한 꼬리 절단 버그가 **Edit 도구 사용 시 2회 재현**

**증상 패턴**:
- 파일 크기: 그대로 유지 (4,761,731 bytes)
- 하지만 내용이 shift되어 **끝 ~61 KB 절단**
- 사라지는 것: `</body>`, `</html>`, `root.render(<BookQuest />)`, serviceWorker 등록 블록, showCoinPopup 나머지
- script 짝: 8/8 → 7/6
- 내가 추가한 `isHiddenTab` 분기는 살아있지만 앱 실행 불가 (JSX 중간 절단)

**복구 절차 (2회 적용)**:
```bash
git show HEAD:index.html > index.html   # 완전 원본 복원
```
```python
# python 바이트 단위 치환 (Edit 우회)
with open('index.html','rb') as f: b = f.read()
assert b.count(OLD_BYTES) == 1
with open('index.html','wb') as f: f.write(b.replace(OLD_BYTES, NEW_BYTES, 1))
```

**원인 추정**: Edit 도구가 4.7 MB 파일에 2 KB 이상 삽입 시 파일 꼬리를 읽지 않고 덮어씀. 문자열 크기와 무관하게 일정 오프셋부터 절단되는 것으로 보임.

- `Edit` 도구로 index.html 수정 **절대 금지** (삽입/삭제 모두 truncation 위험)

### ✅ 권장 워크플로

2. python 바이트 단위 치환으로 수정:
   ```python
   with open('index.html','rb') as f: b = f.read()
   assert b.count(OLD) == 1, "old_string 유일하지 않음"
   new_b = b.replace(OLD, NEW, 1)
   with open('index.html','wb') as f: f.write(new_b)
   ```
3. 매 수정 후 즉시 검증:
   ```bash
   tail -c 100 index.html                   # </html>로 끝나는지
   grep -c '<script' index.html             # open 카운트
   grep -c '</script>' index.html           # close 카운트 (같아야 함)
   wc -c index.html                         # 예상 증분만큼 변했는지
   ```
4. brace/paren diff는 babel 블록 기준으로 체크 (전체 파일은 문자열 리터럴 영향으로 오차)
